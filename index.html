1. Introduction to Terraform

Terraform is an Infrastructure as Code (IaC) tool used to automate cloud and on-prem infrastructure deployment.

terraform init
terraform plan
terraform apply
terraform destroy

Basic Workflow Diagram

Developer Code → terraform init → terraform plan → terraform apply → Infrastructure Created

2. Providers

Providers connect Terraform with cloud platforms like Azure, AWS, and GCP.

provider "azurerm" {
  features {}
  subscription_id = var.subscription_id
}

3. Resources

Resources define actual infrastructure components.

resource "azurerm_resource_group" "rg" {
  name     = "dev-rg"
  location = "Central India"
}

Architecture Diagram

Terraform Code → Azure Provider → Resource Group → VM → Storage → Network

4. Variables

Variables make code reusable and dynamic.

variable "location" {
  default = "Central India"
}

variable "vm_size" {
  type = string
}

5. Outputs

output "resource_group_name" {
  value = azurerm_resource_group.rg.name
}

6. State Management

Terraform stores infrastructure mapping in terraform.tfstate.

terraform state list
terraform state show azurerm_resource_group.rg

7. Modules

Modules help organize reusable infrastructure blocks.

module "network" {
  source = "./modules/network"
}

8. Remote Backend

terraform {
  backend "azurerm" {
    resource_group_name  = "tfstate-rg"
    storage_account_name = "tfstateprod123"
    container_name       = "tfstate"
    key                  = "prod.terraform.tfstate"
  }
}

9. CI/CD Integration

Terraform can integrate with GitHub Actions, Azure DevOps, and Jenkins.

GitHub Push → Pipeline Trigger → terraform init → terraform plan → approval → terraform apply
